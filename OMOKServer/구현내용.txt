4차 진행사항
	- 가끔 vector out_of_range Error 발생...-> 아마 방장클라이언트 죽였을때 그런듯
	- Room을 2개 이상으로 처리하도록 구현할것
	  -> 1. 방을 여러개 생성, 각 각 방장이 들어가 있는 상태로 대기중
                추가로 접속하는 클라리언트가 순차적으로 먼저 생성된 방에 들어가지도록 구현
	  -> 2. 방을 여러개 생성, 각 각 방장이 들어가 있는 상태로 대기중
		추가로 접속하는 클라이언트가 선택해서 들어가는 방식으로 구현

		이중에서 선택해야함.


	- 방장 클라 죽였을때 처리 로직이 없는것 같음 재확인할것 : 이걸 어떻게 확인할것인가...
		-- Check_Alive 패킷 Send : 주기적인 패킷발송(ClientList에 등록되어있는 클라이언트)
		-- STill_Alive 패킷 Recv : 패킷 안들어오면 WSARecv에서 WSAECONNABORTED Error Return을 받아서 Disconnect 처리진행
	  -> 1. 방장만 Room에 있을 경우
	  -> 2. 방장과 다른 클라가 Room에 있을 경우

		위에 부분을 구현하게 되면 전체적이 로직이 다 바뀌도록 처리해야함
